Web Programming Terms

  • DOM (Document Object Model): Think of it as the blueprint of your website that browsers use to build the page. It’s what lets JavaScript talk to your HTML and make cool things happen—like updating content dynamically.
  • API (Application Programming Interface): Fancy name, simple concept. APIs are like the waiters of the web, delivering data between your app and a server when you ask for it.
  • Responsive Design: Making your website look good on any screen, from a widescreen monitor to your phone. It’s all about being adaptable (like a digital chameleon).
  • Semantic HTML: Using meaningful tags like `
    `, `
    `, and `
    ` so your code is more readable for both humans and search engines. It’s like writing with purpose.
  • CSS Grid: The ultimate layout tool for web design. It helps you create precise, clean layouts with rows and columns—kind of like setting up the ultimate Tetris board.
  • JavaScript Events: These are like action triggers for your site. A click, a scroll, or even a mouse hover can spark an event, making the site interactive and alive.
  • GitHub Pages: A free hosting service for your static websites. Think of it as your personal stage to showcase your work to the world.
  • Meta Tags: These live in the `` of your HTML and tell search engines and browsers important details about your site, like its title, description, or which language it’s in.
  • Media Queries: A CSS feature that helps you style your site differently based on the size of the screen. Perfect for making your site mobile-friendly.
  • Favicon: That tiny icon you see in the browser tab for a website. It’s small but mighty—your site’s first impression in the bookmark bar.
